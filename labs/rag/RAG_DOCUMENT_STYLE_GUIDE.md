# RAG 문서 작성 가이드 (Markdown)

이 문서는 검색 품질과 답변 정확도를 높이기 위한 **RAG용 문서 작성 표준**입니다.

## 1) 문서 기본 원칙

- 한 문서는 한 주제만 다룹니다.
- 제목은 검색 키워드를 포함해 구체적으로 작성합니다.
- 문장은 짧고 단정하게 작성합니다.
- 추상 표현보다 명시적 표현을 사용합니다.
- 용어는 문서 전체에서 동일하게 유지합니다.

## 2) 권장 문서 구조

아래 순서를 기본 템플릿으로 사용합니다.

1. 제목
2. 문서 메타데이터
3. 요약
4. 본문 (절차/정책/FAQ 등)
5. 관련 문서
6. 변경 이력

## 3) 메타데이터 규칙

문서 상단에 YAML Front Matter를 사용합니다.

```yaml
---
title: "문서 제목"
doc_type: "guide" # guide | policy | faq | runbook | reference
domain: "rag"
tags: ["rag", "검색", "문서작성"]
owner: "team-name"
version: "1.0.0"
updated_at: "2026-04-13"
source_of_truth: "내부 위키/시스템명"
language: "ko"
---
```

### 메타데이터 작성 팁

- `title`: 질문으로 들어올 표현을 포함합니다.
- `tags`: 동의어/약어를 함께 넣습니다. (예: `["벡터DB", "vector db"]`)
- `updated_at`: 날짜 최신성을 유지합니다.
- `doc_type`: 문서 성격을 명확히 지정합니다.

## 4) 본문 작성 규칙

### 헤더 규칙

- `#`는 문서 제목 1개만 사용합니다.
- 주요 섹션은 `##`, 하위는 `###`로 고정합니다.
- 제목만 보고도 내용을 예측할 수 있게 작성합니다.

### 문장 규칙

- 한 문장에 한 정보만 담습니다.
- 주어를 생략하지 않고 책임 주체를 명시합니다.
- 금지/필수는 각각 `금지`, `필수` 키워드로 통일합니다.

### 목록 규칙

- 나열은 `-` 불릿을 사용합니다.
- 순서가 중요하면 번호 목록을 사용합니다.
- 각 항목은 한 줄 핵심 + 필요 시 하위 설명으로 구성합니다.

### 코드/명령어 규칙

- 명령어는 코드 블록으로 작성합니다.
- 실행 환경(예: `bash`, `python`)을 명시합니다.
- 기대 결과를 한 줄로 첨부합니다.

예시:

```bash
python ingest.py --source ./docs --index faq
```

기대 결과: `faq` 인덱스에 문서가 정상 반영됩니다.

## 5) 검색 친화적 작성법 (RAG 핵심)

- 질문형 소제목을 추가합니다.  
  예: `## 인덱싱 실패 시 무엇을 확인해야 하나요?`
- 동의어/영문 병기를 포함합니다.  
  예: `임베딩(Embedding)`, `재순위화(Reranking)`
- 긴 문서는 의미 단위로 분리합니다. (권장 300~800자)
- 표는 필요한 경우에만 사용하고, 핵심은 본문에도 반복합니다.
- 이미지 의존 설명을 피하고 텍스트로 동일 정보를 제공합니다.

## 6) 금지 사항

- 맥락 없는 축약어만 사용하지 않습니다.
- "이것", "저것", "적당히" 같은 모호 표현을 사용하지 않습니다.
- 최신성이 중요한 값(버전/URL/정책)을 날짜 없이 작성하지 않습니다.
- 하나의 문서에 여러 정책 버전을 혼합하지 않습니다.

## 7) 품질 체크리스트

배포 전 아래 항목을 확인합니다.

- [ ] 문서 상단 메타데이터가 모두 채워져 있습니다.
- [ ] 제목/소제목에 검색 키워드가 포함되어 있습니다.
- [ ] 질문형 쿼리로 검색해도 핵심 답변이 본문에 그대로 존재합니다.
- [ ] 최신 날짜(`updated_at`)가 반영되어 있습니다.
- [ ] 관련 문서 링크가 연결되어 있습니다.

## 8) 바로 쓰는 템플릿

```md
---
title: "문서 제목"
doc_type: "guide"
domain: "rag"
tags: ["키워드1", "키워드2", "동의어"]
owner: "team-name"
version: "1.0.0"
updated_at: "YYYY-MM-DD"
source_of_truth: "시스템/위키"
language: "ko"
---

# 문서 제목

## 요약
- 이 문서가 답하는 질문 1문장
- 적용 대상/범위 1문장

## 배경
- 왜 이 문서가 필요한지

## 절차
1. 사전 조건
2. 실행 단계
3. 검증 방법

## 자주 묻는 질문 (FAQ)
### 질문 1
- 답변

### 질문 2
- 답변

## 트러블슈팅
### 증상
- 관찰되는 현상

### 원인
- 가능한 원인

### 해결
- 조치 방법

## 관련 문서
- [문서명](./관련문서.md)

## 변경 이력
- 2026-04-13: 초안 작성
```
